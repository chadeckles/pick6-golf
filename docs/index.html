🏌️ Pick Six Golf

A full-stack pick'em pool app for golf's four major championships. Draft 6 golfers across skill tiers, and your best 5 of 6 to-par scores decide the winner — with live scoring, no spreadsheets.

View the code on GitHub → See how it's built

What it is

Pick Six Golf is a self-hosted web app for running major-championship pools with friends. I built it to sharpen my full-stack skills — real authentication, a live third-party data feed, and a scoring engine with actual edge cases. It's kept here as a code sample; it is not currently deployed to a live host.

🎯

Tiered drafting

Pick 6 golfers across 4 tiers ranked by Official World Golf Rankings — 1-2-2-1 format.

📡

Live ESPN scoring

Real-time leaderboard pulled from ESPN's public scoreboard API. No manual score entry.

🧮

Best 5 of 6

Your worst golfer is dropped. Cut penalties, tiebreakers, and to-par math all handled server-side.

🎨

Dynamic theming

Each tournament — Masters, PGA, US Open, The Open — gets its own palette and branding automatically.

🔒

Real auth & security

bcrypt password hashing, JWT sessions, rate limiting, CSRF origin checks, and input validation.

💰

Pool administration

Invite codes, payment tracking, configurable pick-lock dates, and admin transfer.

How a pool works

The whole experience in five steps.

  1. Create a pool — you get an invite code to share.
  2. Friends join — they register and enter the code.
  3. Pick 6 golfers across 4 OWGR-ranked tiers before picks lock.
  4. Watch the live leaderboard — scoring updates automatically during the tournament.
  5. Best 5 of 6 wins — one built-in mulligan, because we all need one.

How it's built

A server-rendered full-stack app — chosen deliberately over a static site so I could work with a real database, sessions, and API routes.

Engineering highlights

The parts I'm proud of — the things that made it a genuine learning project rather than a tutorial clone.

Locked tier snapshots

Rankings are frozen the moment a tournament's field is built, so every member drafts from the exact same pool no matter when they pick.

Data pipeline

A YAML schedule and live OWGR data feed a set of Node scripts that generate immutable, version-controlled tier tables.

Scoring engine

To-par math, missed-cut penalties, "drop your worst" logic, and tiebreakers — all with server-side validation.

Security-first API

Every route guards against the usual suspects: hashed passwords, signed sessions, rate limits, and origin checks.

Run it yourself

cd pick6-golf
npm install
